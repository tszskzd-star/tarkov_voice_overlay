# Signaling protocol

The coordinator accepts WebSocket clients on `/ws`. All messages are JSON.

## Envelope

```json
{
  "type": "ice_candidate",
  "roomId": "room_abcd",
  "peerId": "peer_sender",
  "targetPeerId": "peer_receiver",
  "nick": "Player",
  "payload": {},
  "sentAt": 1781580000000
}
```

## Client to server

- `hello`: sets/updates `nick`.
- `list_rooms`: requests public room snapshots.
- `create_room`: creates a room. `payload` may contain host public key and room
  salt. No password is sent.
- `join_room`: joins a room. `payload` may contain joining public key and local
  password proof.
- `leave_room`: leaves the current room.
- `ice_offer`, `ice_answer`, `ice_candidate`: routed to `targetPeerId`. The
  payload carries direct voice transport candidates, currently UDP host/STUN
  candidates generated by the client.
- `mute_status`, `peer_status`: broadcast inside the room when no
  `targetPeerId` is provided.
- `voice_frame`: rejected. Voice data must be sent directly between clients.
- `heartbeat`: answered with `heartbeat_ack`.

## Server to client

- `hello`: includes the assigned `peerId`.
- `rooms`: contains public room snapshots.
- `room_created`: confirms room creation.
- `joined_room`: confirms join and includes full public room snapshot.
- `peer_joined`, `peer_left`: room membership events.
- `room_closed`: host left and the room was removed.
- `error`: protocol or capacity issue.

Voice data must never be sent through the coordinator. The coordinator is only
allowed to carry room metadata, peer status, and signaling messages.
